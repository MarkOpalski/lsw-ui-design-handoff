Design Tokens

Design tokens are the visual design atoms of the design system. They store visual design attributes and replace hard-coded values in our codebase.

Available Formats

  • CSS Custom Properties: design-tokens.css
  • TypeScript Constants: design-tokens.ts
  • Design Tokens Community Format: design-tokens.json 
/* CSS Usage */
.button {
  background-color: var(--color-primary-600);
  padding: var(--spacing-sm) var(--spacing-md);
  border-radius: var(--radius-md);
}

/* TypeScript Usage */
import { colors, spacing, borderRadius } from './tokens/design-tokens';

const buttonStyles = {
  backgroundColor: colors.primary[600],
  padding: `${spacing.sm} ${spacing.md}`,
  borderRadius: borderRadius.md
};

Usage Guidelines

Best practices and implementation guidelines for using the design system.

Getting Started

  1. Import the design tokens CSS file or TypeScript constants
  2. Use design tokens instead of hard-coded values
  3. Follow component patterns for consistency
  4. Test accessibility and responsiveness

Design Token Usage

  • Colors: Always use semantic color tokens (primary, success, error) rather than specific hue values
  • Spacing: Use the spacing scale for consistent margins, padding, and gaps
  • Typography: Follow the type scale for headers, body text, and UI elements
  • Shadows: Apply consistent elevation with predefined shadow tokens

Component Guidelines

  • Buttons: Use primary for main actions, secondary for alternative actions
  • Messages: Match message type to content severity (info, success, warning, error)
  • Accessibility: Include proper ARIA labels and focus states
  • Responsive: Test components at different screen sizes
